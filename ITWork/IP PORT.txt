1. 공인 IP / 내부 IP
공인 IP는 인터넷상에서 겹치지 않는 세상에서 하나씩 뿐인 IP,
전세계 어디에서나 해당 IP주소로 접근 가능
보통 ISP(인터넷 서비스 제공자)를 통해서 할당받음

해저케이블 설치 되어있는 걸 볼 수 있는 사이트
https://www.submarinecablemap.com/

내부IP(사설, 가상 IP)
특정 네트워크 안에서만 유효한 IP(보통 와이파이, 공용 컴퓨터, 공유기를 사용하는 곳),
특정범위의 IP주소들이 사설 IP주소로 만들어져있음.
(예: 192.168.x.x, 10.x.x.x, 172.16.x.x ~ 172.31.x.x).
이렇게 안쓰고 자기가 임의로 설정해서 쓸 수도 있음

고정 IP / 유동 IP
고정 IP는 말그대로 한 장치에 고정적으로 할당되는 IP 주소.
보통 일반적으로는 대부분 유동 IP로 사용하고 있고,
DHCP를 통해서 동적으로 IP주소를 할당받음.

DHCP(동적 호스트 설정 프로토콜)
내부IP에서 고정된 IP를 제외하고,
IP를 필요로 하는 컴퓨터에 자동으로 할당하고, 사용하지 않으면 다시 반환받아서,
다른 컴퓨터도 그 IP를 사용할 수 있도록 함.
그래서 와이파이 같은 거는 사용하다가 끊고 다시 사용하게 되면 IP가 바뀜.

서브넷마스크
IPv4 주소의 고갈을 최대한 늦추기 위해
각 라우터가 브로드캐스팅하는 로컬 네트워크 영역에 공인 IP 대역을 호스트가 
필요한 만큼만 할당하려는 노력을 하였다.
이러한 NIC 기관의 요구에 맞춰서 IETF에서는 로컬 네트워크

내부에서 접속한 호스트의 IP 대역을 외부 네트워크와 명확하게
구분할 수 있는 수단을 표준화하였고 이것이 서브넷 마스크(Subnet Mask)다.

서브넷 마스크의 기본 원칙:
서브넷 마스크는 연속된 1들로 시작합니다.
1들의 끝나는 지점 다음에는 오로지 0만 있어야 합니다. 즉, 1과 0이 번갈아 나타나는 패턴 (예: 11001010)은 유효한 서브넷 마스크가 아닙니다.
서브넷 마스크의 이진 표현:
IPv4 주소는 32비트로 구성됩니다. 서브넷 마스크도 동일하게 32비트로 표현됩니다.

예를 들어, 서브넷 마스크 255.255.255.128은 이진수로 다음과 같이 표현됩니다:

11111111.11111111.11111111.10000000

위의 이진 표현에서 볼 수 있듯이, 처음 25비트는 1이고 나머지 7비트는 0입니다. 이는 /25로 표현될 수 있습니다.

서브넷 마스크 변경:
네트워크에서 필요한 호스트 수에 따라 서브넷 마스크를 변경할 수 있습니다.

만약 더 많은 호스트가 필요하다면, 서브넷 마스크에 있는 1의 수를 줄입니다. 예를 들어, /25 (255.255.255.128)에서 /24 (255.255.255.0)로 변경하면, 사용 가능한 호스트 수가 2배로 증가합니다.
반대로, 더 많은 서브넷이 필요하고 각 서브넷에서의 호스트 수는 줄일 수 있다면, 서브넷 마스크에 있는 1의 수를 늘립니다.
이러한 변경을 할 때는 네트워크의 구조, 향후 확장성, IP 주소의 효율적인 사용 등 여러 요인을 고려해야 합니다.


기본 게이트웨이 (Default Gateway)는 네트워크에서 로컬 네트워크 (LAN)의 경계에 위치하며, 
로컬 네트워크에 있는 장치들이 외부 네트워크나 인터넷으로 통신할 때 사용하는 
중간 라우터나 스위치를 가리킵니다.

간단히 말해서, 컴퓨터나 다른 장치가 자신이 속한 로컬 네트워크 외부의 
다른 네트워크에 데이터를 보내려고 할 때, 해당 데이터 패킷은 기본 게이트웨이를 통해 라우팅됩니다. 
기본 게이트웨이는 이러한 패킷을 적절한 목적지로 전달하는 역할을 합니다.

예를 들어, 집에서 Wi-Fi를 사용하여 인터넷을 사용할 때, 
보통의 홈 라우터는 LAN의 기본 게이트웨이 역할을 합니다. 
만약 웹 브라우저를 사용해 웹사이트에 접속하려고 하면, 
해당 요청은 먼저 홈 라우터 (기본 게이트웨이)로 전달되고, 
홈 라우터는 인터넷 서비스 제공자 (ISP)를 통해 웹사이트의 서버로 요청을 전달합니다.

기본 게이트웨이는 일반적으로 네트워크 설정의 일부로 장치에 설정됩니다.
 DHCP를 사용하는 네트워크에서는, DHCP 서버가 장치에 IP 주소와 함께 
 기본 게이트웨이의 주소를 자동으로 제공합니다.
